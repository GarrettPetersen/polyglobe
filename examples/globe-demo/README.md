# Globe demo

Run `npm run dev` and open the app. Use Earth map mode to see real geography.

## Data (Earth mode)

- **Land/sea**: Loaded from world-atlas (TopoJSON). Major lakes (Great Lakes, Baikal, Victoria, etc.) are punched out from land via Natural Earth 110m lakes GeoJSON so they render as water.
- **Climate (deserts, rainforest)**: Loaded from `public/koppen.bin` (360×180 Köppen–Geiger). Generated by `npm run build-koppen` (already run; file is committed). Without it, a latitude fallback is used (subtropical = desert, tropics = land).
- **Elevation (mountains)**: Optional. If `public/elevation.bin` exists (360×180 float32, meters, row 0 = north), it is used so tiles above ~1500 m become mountains. Run `npm run build-elevation` to create a placeholder; replace with real ETOPO1/SRTM data for real relief.
- **3D peaks**: `public/mountains.json` is generated from the **Natural Earth 110m geography_regions_elevation_points** dataset (mountains only). Run `npm run build-mountains` to fetch it and write the file. The 110m dataset has a small set of major peaks; for more, use another source (e.g. [GeoNames](https://download.geonames.org/export/dump/)) and output the same JSON format: `[{ "name", "lat", "lon", "elevationM" }, ...]`.
